### Capacitor 설치 및 초기화
`npm install @capacitor/core@8`
`npm install -D @capacitor/cli@8`
- @8은 버전을 직접적으로 지정, 제거 시 가장 최신 버전 설치
  
`npx cap init`
- Capacitor 설정 파일 생성
  App name: 앱 이름
  App Package Id: 앱 고유 식별자, 보통 도메인을 거꾸로 씀
  Web asset directory: 웹 빌드 결과물 저장된 폴더명

네이티브 프로젝트 생성
`npm install @capacitor/android @capacitor/ios`
`npx cap add android`
`npx cap add ios`

### 웹과 네이티브 동기화
웹 코드 수정시 반드시 dist 폴더의 내용물을 android와 ios 폴더 안으로 복사해 주는 작업을 해야 한다.
`npx cap sync`
### Capacitor 네이티브 & 웹 분기처리
최신 capacitor에는 코어 모듈 자체에 플랫폼 판별기를 내장하고 있다.
`
import { Capacitor } from '@capacitor/core'
cosnt isApp = Capacitor.isNativePlatform();
const platform = Capacitor.getPlatform()
`
-> 웹프로젝트 util에 생성 후 함수에서 분기처리

### Capacitor 플러그인
#### 1. 햅틱 (Haptics)
`npm install @capacitor/haptics`
`npx cap sync`

- 사용 예시 (`src/utils/capacitorModule.ts`)
```typescript
import { Haptics, ImpactStyle } from '@capacitor/haptics'

export const nativeHaptic = async () => {
    await Haptics.impact({ style: ImpactStyle.Heavy })
}
```

#### 2. Status bar
- 상태바 관련 앱 초기 설정은 `capacitor.config.ts` 에서 설정을 해줘야 한다
```typescript
plugins: {
  StatusBar: {
    visible: true,
    overlaysWebView: false 
  }
}
```

- **안드로이드 15 (API 35+) 테마 설정 (`styles.xml`)**
최신 안드로이드 버전(15+)에서는 Edge-to-Edge 디스플레이 설정이 강제되어 상태바 영역까지 앱이 차지하면서 상태바가 투명해지거나 동작하지 않는 현상이 발생합니다. 이를 막으려면 `styles.xml`에 예외 처리를 추가해야 합니다.
```xml
<style name="AppTheme.NoActionBar" parent="Theme.AppCompat.DayNight.NoActionBar">
    ...
    <item name="android:windowFullscreen">false</item> <!-- 풀스크린 해제 -->
    <item name="android:windowOptOutEdgeToEdgeEnforcement">true</item> <!-- Android 15 Edge-to-Edge 무효화 (상태바 강제 표시) -->
</style>

<style name="AppTheme.NoActionBarLaunch" parent="Theme.SplashScreen">
    <item name="android:background">@drawable/splash</item>
    <item name="postSplashScreenTheme">@style/AppTheme.NoActionBar</item> <!-- 스플래시 후 테마 전환 필수 -->
</style>
```

`npm install @capacitor/status-bar`
`npx cap sync`
